## Reporting a Vulnerability

If you discover a security vulnerability or suspect a security issue, **please do not open a public GitHub issue.**

Instead, contact me directly:

- **Email:** arkbarua16@gmail.com  
- **GitHub Profile:** https://github.com/Ark-Barua

I will review your report within **48 hours**.

### What to Include
Please provide as much detail as possible:

- Description of the vulnerability  
- Steps to reproduce  
- Expected behavior  
- Actual behavior  
- A proof-of-concept (PoC), if possible  
- Any potential impact or risk

### Responsible Disclosure
I follow a responsible disclosure process:

1. You report the issue privately  
2. I investigate and confirm the vulnerability  
3. A patch is prepared and tested  
4. A new release is published  
5. You may optionally be credited for the discovery  

### Do Not
- Do **not** publicly disclose the vulnerability before it is fixed  
- Do **not** exploit any vulnerabilities  
- Do **not** perform destructive testing against live systems  

## Security Best Practices for Users
To help keep your instance secure:

- Keep dependencies updated  
- Never expose `.env` files or secrets publicly  
- Use environment variables for API keys  
- Rotate credentials periodically  
- Host the backend on a secure HTTPS environment (Render generates HTTPS automatically)
